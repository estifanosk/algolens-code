# Greedy

Each implementation exposes a brute-force or unoptimized method next to the optimized greedy method. The optimized method uses the LeetCode-style name from the problem prompt.

## Problems

- `assign-cookies`
- `lemonade-change`
- `jump-game-ii`
- `gas-station`
- `can-place-flowers`
- `queue-reconstruction-height`
- `partition-labels`
- `valid-parenthesis-string`
- `hand-of-straights`
- `minimum-add-parentheses`

## Files

- Java: `java/src/main/java/algolens/greedy/GreedyAlgorithms.java`
- Python: `python/greedy/algorithms.py`
- TypeScript: `typescript/src/greedy/greedy-algorithms.ts`

## Run

Java:

```bash
cd java
mvn test
mvn exec:java -Dexec.mainClass=algolens.greedy.GreedyMain
```

Python:

```bash
cd python
pytest
python -m greedy
```

TypeScript:

```bash
cd typescript
npm run typecheck
npm test
```
